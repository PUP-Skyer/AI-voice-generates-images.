# AI语音控制绘图工具 -- 设计文档

> 版本：v1.0 | 日期：2026-06-14

---

## 1. 项目概述

### 1.1 项目背景

本项目旨在开发一款**纯语音控制的绘图工具**，用户完全通过语音指令完成绘图创作，不使用鼠标或键盘。工具同时支持两种绘图模式：

1. **语音驱动Canvas画布绘图**：用户通过语音指令控制在HTML5 Canvas上绘制基本图形
2. **语音调用AI生成图片**：用户通过语音描述场景，调用外部AI API生成完整图片

### 1.2 核心约束

- 用户不能使用鼠标或键盘进行绘图操作（仅语音）
- 需要考虑指令理解的准确性与容错性
- 需要考虑语音到绘图操作的响应延迟
- 需要考虑复杂指令的拆解与执行能力

### 1.3 技术选型

| 技术 | 选择 | 原因 |
|------|------|------|
| 文件格式 | 单HTML文件 | 方便分享和使用 |
| 样式框架 | Tailwind CSS (CDN) | 快速开发、与参考文件一致 |
| 语音识别 | Web Speech API | 浏览器内置、免费、无需后端 |
| 语音合成 | SpeechSynthesis API | 用于操作确认反馈 |
| 绘图引擎 | HTML5 Canvas 2D | 原生API、性能好、兼容性好 |
| 数据缓存 | IndexedDB | 大容量本地存储 |
| 状态持久化 | localStorage | 轻量级配置存储 |


## 3. 语音识别模块

### 3.1 可插拔适配器设计

采用**适配器模式（Adapter Pattern）**，定义统一的语音识别接口：

```javascript
class SpeechAdapter {
  start()                    // 开始识别
  stop()                     // 停止识别
  setLanguage(lang)          // 设置语言
  isSupported()              // 检查支持
  // 回调
  onResult(text, isFinal)    // 识别结果
  onError(error)             // 错误处理
  onStart()                  // 开始事件
  onEnd()                    // 结束事件
}
```

当前实现：`WebSpeechAdapter`（基于浏览器内置API）

预留接口：
- `IFlytekSpeechAdapter`（讯飞语音）
- `BaiduSpeechAdapter`（百度语音）

### 3.2 Web Speech API 配置

| 参数 | 值 | 说明 |
|------|------|------|
| `lang` | `zh-CN` | 简体中文 |
| `continuous` | `true` | 持续识别不停顿 |
| `interimResults` | `true` | 实时返回中间结果 |
| `maxAlternatives` | `3` | 多候选，提高容错率 |

### 3.3 连续识别与自动重启

Web Speech API在`continuous=true`模式下，静默超时后可能自动停止。解决方案：

- `onend`事件：检测`_isListening`状态，自动调用`adapter.start()`重启
- `no-speech`/`aborted`错误：静默重启（延迟200ms）
- 其他错误：显示Toast通知用户

### 3.4 第三方API扩展接口

要接入第三方语音API，需要：

1. 创建新的Adapter类继承`SpeechAdapter`
2. 实现`start()`、`stop()`、`setLanguage()`、`isSupported()`方法
3. 在`onResult`回调中返回`{ text, isFinal }`格式的结果
4. 在VoiceDrawApp中切换adapter实例

### 4.3 容错策略

| 策略 | 实现方式 | 效果 |
|------|---------|------|
| **同义词覆盖** | 每个意图配置5-10个同义词/近义词正则 | 覆盖常见表达方式 |
| **多候选利用** | maxAlternatives=3，首选置信度低于阈值时尝试第二、第三候选 | 提高识别准确率 |
| **上下文继承** | 上一条指令设置的颜色/线宽，后续指令未指定时自动沿用 | 减少重复指令 |
| **部分指令补全** | 用户只说"红色"，无操作动词时默认继承上一次绘制类型 | 支持简短指令 |
| **指令确认** | 置信度<0.5时，用SpeechSynthesis反问确认 | 防止误操作 |
| **指令纠错日志** | 记录未识别指令到日志面板 | 方便调试和优化 |
| **同音字容错** | 过滤常见无关词（如"花园"→"画"+"圆"） | 中文语音特有问题 |

## 5. Canvas绘图引擎

### 5.1 命令模式(Command Pattern)

每一步绘图操作封装为`DrawCommand`对象：

```javascript
class DrawCommand {
  type: string        // 'line' | 'rect' | 'circle' | ...
  params: object      // 图形参数
  style: object       // 样式参数
  execute(ctx)        // 执行绘制
}
```

优点：
- 天然支持撤销/重做（将命令从undoStack移到redoStack）
- 每个命令可独立执行和序列化
- 新增图形类型只需添加新的_draw方法

### 5.2 支持的图形列表

| 图形 | type | 核心参数 | Canvas API |
|------|------|---------|------------|
| 直线 | `line` | x1,y1,x2,y2 | moveTo + lineTo |
| 矩形 | `rect` | x,y,w,h | strokeRect / fillRect |
| 圆形 | `circle` | cx,cy,r | arc(0, 2π) |
| 椭圆 | `ellipse` | cx,cy,rx,ry | ellipse() |
| 三角形 | `triangle` | points[3] | moveTo + lineTo × 3 |
| 多边形 | `polygon` | cx,cy,r,sides | 三角函数计算顶点 |
| 星形 | `star` | cx,cy,outerR,innerR,points | 内外半径交替 |
| 箭头 | `arrow` | x1,y1,x2,y2 | 线 + 三角箭头 |
| 文字 | `text` | text,x,y | fillText / strokeText |
| 自由绘制 | `freehand` | points[] | 连续lineTo |

### 5.3 智能定位策略

语音指令通常不会指定精确坐标。通过位置关键词映射到画布百分比坐标：

| 关键词 | X% | Y% | 说明 |
|--------|-----|-----|------|
| 左上 | 20% | 20% | 左上区域 |
| 右上 | 80% | 20% | 右上区域 |
| 左下 | 20% | 80% | 左下区域 |
| 右下 | 80% | 80% | 右下区域 |
| 中间 | 50% | 50% | 画布中心 |
| 左边 | 25% | 50% | 左侧中心 |
| 右边 | 75% | 50% | 右侧中心 |
| 上边 | 50% | 25% | 上方中心 |
| 下边 | 50% | 75% | 下方中心 |

未指定位置时默认在画布中心(50%, 50%)绘制。

### 5.4 撤销/重做机制

- **最大历史记录**：50步
- **新操作**：清空重做栈
- **撤销**：将最后一个命令从undoStack移到redoStack，然后完整重绘
- **重做**：将最后一个命令从redoStack移到undoStack，然后完整重绘
- **完整重绘**：清空画布 → 重绘背景 → 依次执行所有历史命令

## 6. AI图片生成

### 6.1 API集成方案

- **API地址**：`https://api.wuyinkeji.com/api/async`
- **支持模型**：
  - `image_gpt`：GPT模型（0.1元/张，不支持本地图）
  - `image_nanoBanana_pro`：NanoBanana Pro（0.3元/张）
- **API Key**：存储在localStorage，UI提供配置入口

### 6.2 异步轮询机制

```
用户语音 "生成一张猫的图片"
    ↓
提交任务到 API → 获取 taskId
    ↓
每3秒轮询查询状态
    ↓
状态=completed → 获取图片URL → 加入画廊
状态=failed → 显示错误Toast
超时(120秒) → 显示超时Toast
```

### 6.3 图片缓存（IndexedDB）

- **数据库名**：`VoiceDrawCache`
- **存储结构**：`{ taskId, url, prompt, timestamp }`
- **用途**：离线查看已生成的图片，支持批量下载

### 6.4 图片画廊管理

- 右侧栏显示所有生成的图片缩略图
- 点击打开灯箱预览
- 灯箱支持：缩放（滚轮）、平移（拖拽）、左右切换

## 9. 语音反馈系统

### 9.1 语音合成(SpeechSynthesis)确认

当指令置信度低于0.5时，使用语音合成反问确认：

```javascript
_speakConfirm(command) {
  const confirmMap = {
    'drawCircle': '您是要画圆形吗？',
    'drawRect': '您是要画矩形吗？',
    'drawLine': '您是要画直线吗？',
    // ...
  };
  this._speak(confirmMap[command.action] || '请再说一次');
}
```

### 9.2 Toast通知

四种类型：info（蓝色）、success（绿色）、warning（黄色）、error（红色）

## 13. 创新特性：实时语音Canvas预览

### 13.1 创新概念

传统语音绘图工具的工作流程是：**用户说完 → 系统解析 → 执行绘制**。用户无法在说话过程中看到即将绘制的内容，存在"黑箱感"。

本工具实现了**实时Canvas预览（Live Preview）**：用户说话的同时，Canvas上实时出现半透明的"幽灵"图形预览。当语音指令最终确认时，预览"凝固"为正式图形，同时伴随白色闪光确认效果。

```
传统流程:  [说话] → [等待] → [解析] → [绘制]
本工具:    [说话的同时Canvas实时显示预览] → [确认时凝固为正式图形 + 闪光]
```

### 13.2 技术架构

```
┌──────────────────────────────────────────────────────────────┐
│                   Web Speech API                              │
│  interimResults=true → 每次识别变化都触发回调                   │
├──────────────────────────────────────────────────────────────┤
│              isFinal=false (中间结果)                          │
│  ┌─────────────┐    ┌──────────────┐    ┌────────────────┐  │
│  │InterimParser │ →  │ LivePreview  │ →  │ 预览Canvas      │  │
│  │(实时解析)     │    │ (幽灵渲染)   │    │ (半透明虚线)    │  │
│  └─────────────┘    └──────────────┘    └────────────────┘  │
├──────────────────────────────────────────────────────────────┤
│              isFinal=true (最终结果)                           │
│  ┌─────────────┐    ┌──────────────┐    ┌────────────────┐  │
│  │LivePreview   │ →  │ 闪光过渡效果  │    │ 预览Canvas清除  │  │
│  │._commitPreview│   │              │    │                │  │
│  └─────────────┘    └──────────────┘    └────────────────┘  │
│  ┌─────────────┐    ┌──────────────┐    ┌────────────────┐  │
│  │CommandParser │ →  │ CanvasEngine │ →  │ 主Canvas正式绘制│  │
│  │(完整解析)     │    │ (执行绘制)   │    │ (实线+填充)    │  │
│  └─────────────┘    └──────────────┘    └────────────────┘  │
└──────────────────────────────────────────────────────────────┘
```

### 13.3 新增模块

| 模块 | 类名 | 职责 |
|------|------|------|
| 中间文本解析器 | `InterimParser` | 从语音中间文本实时提取形状、颜色、位置、大小等参数 |
| 实时预览引擎 | `LivePreview` | 在预览Canvas上渲染半透明"幽灵"图形，支持动态效果 |
| 预览Canvas | `previewCanvas` | 叠加在主Canvas上方的透明层，用于预览渲染 |

### 13.4 InterimParser 核心设计

**增量解析**：不需要完整指令，从部分文本中提取可用参数。

```javascript
// 用户说"画一个红" → 检测到形状=圆(未确定), 预览显示默认圆形+红色
// 用户说"画一个红色的圆" → 检测到形状=圆, 颜色=红色, 预览更新
// 用户说"画一个红色的大圆在左上角" → 所有参数确定, 预览完整显示
```

检测维度：
- **形状**：画圆/画矩形/画线/画三角... (9种)
- **颜色**：红/蓝/绿/黄/黑/白/紫/橙/粉/灰/棕/青 (12种)
- **位置**：左上/右上/左下/右下/中间/左边/右边/上边/下边 (9个)
- **大小**：小/中/大 + 数字 (如"线宽5")
- **动态变化**：越来越大/越来越小/旋转/越来越淡 (4种)

### 13.5 LivePreview 渲染特性

| 特性 | 实现 |
|------|------|
| 半透明显示 | `globalAlpha = 0.4` |
| 虚线边框 | `setLineDash([6, 4])` |
| 中心十字标记 | 帮助用户确认图形位置 |
| 端点圆点 | 直线/箭头的端点标记 |
| 动态增长 | "越来越大" → requestAnimationFrame实时更新大小 |
| 闪光确认 | isFinal时短暂白色闪烁(150ms) |
| 状态指示器 | 右上角显示"预览: 圆形"等标签 |

### 13.6 防重复绘制机制

预览Canvas和主Canvas分离，避免重复绘制：
- **isFinal=false**：只在预览Canvas渲染，主Canvas不变
- **isFinal=true**：预览Canvas清除 → 主Canvas执行正式绘图
- `_commitPreview()`只做闪光效果，不调用主引擎绘图方法
- `_processCommand()`负责真正的Canvas绘制
